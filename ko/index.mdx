Lynx는 AI 에이전트 실행 중에 무슨 일이 있었는지 나중에 확인할 수 있게 도와줍니다. SDK를 연결하면 세션, 실행 단계, LLM 호출, 도구 호출, 오류, 가드레일 판단을 하나의 흐름으로 기록합니다.

Lynx는 단순히 로그를 많이 남기는 도구가 아닙니다. 운영 중인 AI 서비스에서 문제가 생겼을 때 어떤 입력, 컨텍스트, 판단, 도구 호출이 영향을 줬는지 빠르게 좁혀갈 수 있게 만듭니다.

## Lynx가 기록하는 정보

SDK는 다음 정보를 기록할 수 있습니다.

- 세션, 실행, 워크스페이스, 에이전트, 환경 메타데이터
- 사용자 입력과 선택된 컨텍스트
- 에이전트 판단과 그 이유
- LLM 호출, 모델 정보, 지연 시간, 토큰 사용량, 오류
- 도구 호출, 도구 결과, 부작용 여부, 위험도
- 로컬 정책 판단과 가드레일 동작
- 메모리 접근과 최종 실행 결과

## Lynx에서 확인할 수 있는 것

이벤트가 전송되면 Lynx에서 다음 정보를 확인할 수 있습니다.

- 전체 세션 타임라인
- 각 LLM 호출과 도구 호출
- 지연 시간, 토큰 사용량, 예상 비용
- 오류, 재시도, 느린 실행 단계
- 가드레일 판단과 차단된 도구 호출
- 관련 근거가 있는 원인 후보

## 운영 환경에서의 동작

기본 전송 방식은 백그라운드 전송입니다. Lynx API에 일시적으로 접근할 수 없어도 고객 서비스는 계속 실행됩니다. SDK는 이벤트를 메모리 큐에 보관하고, 전송 실패를 다시 시도하며, 실패가 반복되면 서킷 브레이커를 엽니다.

가드레일 판단은 프로세스 안에서 로컬로 실행됩니다. Lynx API 상태가 도구 호출 차단 여부를 결정하지 않습니다.

<Note>
  SDK 설정에 따라 프롬프트, 도구 인자, 모델 응답이 Lynx로 전송될 수
  있습니다. API 키, 토큰, 비밀번호 같은 민감정보는 전송 전에 마스킹하는
  것을 권장합니다. 원문을 보내고 싶지 않다면 `metadata-only` 모드를
  사용하세요.
</Note>
