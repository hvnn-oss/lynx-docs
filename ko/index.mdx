Lynx SDK는 AI 에이전트가 왜 특정 행동을 했는지 이해할 수 있게 해줍니다. 실행 컨텍스트, LLM 호출, 도구 사용, 정책 결정, 메모리 접근, 최종 결과를 수집해 Lynx 서버로 전송합니다.

Lynx는 단순 로그가 아니라 프로덕션 AI 서비스의 관측 가능성을 위해 설계되었습니다.

## Lynx가 수집하는 정보

- Agent run, session, workspace 메타데이터
- 사용자 입력과 구조화된 context
- 에이전트 결정과 그 이유
- LLM 호출, 모델 메타데이터, latency, token usage, error
- Tool call, tool result, side effect, risk metadata
- 로컬 policy evaluation과 guardrail activation
- Memory access와 session outcome

## Lynx 서버가 내려가면 어떻게 되나요?

기본 설정에서는 AI 서비스가 계속 실행됩니다. SDK는 telemetry를 메모리에 queueing하고, 전송 실패 시 retry하며, 반복 실패 후 circuit breaker를 엽니다. Guardrail 결정은 계속 로컬에서 수행됩니다.

<Tip>
  Lynx telemetry는 관측을 위한 것이며 기본적으로 애플리케이션 실행을 막지 않습니다. 필요한 환경에서는 blocking flush를 명시적으로 켤 수 있습니다.
</Tip>
