Lynx는 telemetry 전송 실패가 AI 서비스를 멈추지 않도록 설계되었습니다.

## 프로덕션 기본값

| 옵션 | 기본값 |
| --- | --- |
| `delivery.mode` | `"BACKGROUND"` |
| `delivery.timeoutMs` | `1000` |
| `delivery.flushOnRunEnd` | `false` |
| `delivery.flushIntervalMs` | `3000` |
| `delivery.batchSize` | `50` |
| `delivery.maxQueueSize` | `1000` |
| `delivery.overflowStrategy` | `"DROP_OLDEST"` |
| `circuitBreaker.enabled` | `true` |
| `circuitBreaker.failureThreshold` | `3` |
| `circuitBreaker.cooldownMs` | `30000` |

## Background delivery

Background mode에서 `run()`은 event를 로컬에 기록하고 telemetry 전송을 기다리지 않습니다. SDK는 timer, 명시적 `flush()`, 또는 `shutdown()`에서 event를 전송합니다.

```ts
const lynx = new LynxTracer({
  clientId: "support-api",
  endpoint: "https://lynx.example.com",
  apiKey: process.env.LYNX_API_KEY,
  delivery: {
    mode: "BACKGROUND",
    timeoutMs: 1000,
    flushOnRunEnd: false,
  },
});
```

## Blocking delivery

현재 request lifecycle 안에서 전송 시도를 해야 하는 환경에서만 blocking mode를 사용하세요.

```ts
const lynx = new LynxTracer({
  clientId: "worker",
  endpoint: "https://lynx.example.com",
  apiKey: process.env.LYNX_API_KEY,
  delivery: {
    mode: "BLOCKING",
    flushOnRunEnd: true,
    timeoutMs: 1500,
  },
});
```

## Flush와 shutdown

```ts
await lynx.flush();
```

프로세스가 종료되기 전에는 `shutdown()`을 사용하세요.

```ts
await lynx.shutdown({ timeoutMs: 1000 });
```

## 로컬 상태 확인

```ts
const status = lynx.getStatus();

console.log(status.queueSize);
console.log(status.circuitState);
console.log(status.droppedEvents);
```

## Lynx 서버가 내려간 경우

SDK는 network error, timeout, non-2xx response를 catch합니다. 실패한 batch는 메모리 retry queue로 이동합니다. 반복 실패 후 circuit breaker가 열리고 cooldown이 끝날 때까지 전송 시도를 멈춥니다.

Queue가 `maxQueueSize`에 도달하면 SDK는 `overflowStrategy`에 따라 event를 drop합니다.
