title: "설정하기"
description: "Lynx SDK의 endpoint, 수집 모드, 전송 방식, 환경 메타데이터를 설정합니다."
---

SDK 설정으로 이벤트를 어디로 보낼지, 어떤 내용을 기록할지, 운영 환경에서 전송 실패를 어떻게 다룰지 정할 수 있습니다.

## 언제 사용하나요

다음 상황에서 이 문서를 참고하세요.

- Lynx Cloud 또는 로컬 백엔드로 이벤트를 보내야 할 때
- 워크스페이스, 에이전트, 버전, 환경 정보를 붙여야 할 때
- 프롬프트나 응답 원문을 보내고 싶지 않을 때
- 전송 timeout, 큐, 서킷 브레이커를 조정해야 할 때

## 기본 설정

```ts
import { LynxTracer } from "@lynxops/sdk";

const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
  workspaceId: "workspace_123",
  environment: "production",
  appVersion: "1.4.0",
});
```

`endpoint` 기본값은 `https://api.lynxops.co`입니다.

로컬 개발이나 자체 서버를 사용할 때만 endpoint를 직접 지정합니다.

```ts
const lynx = new LynxTracer({
  clientId: "support-api",
  endpoint: "http://localhost:4000",
  apiKey: process.env.LYNX_API_KEY,
});
```

## 수집 옵션

| 옵션 | 기본값 | 설명 |
| --- | --- | --- |
| `sampleRate` | `1` | 기록할 실행 비율입니다. `0`부터 `1`까지 설정합니다. |
| `captureInput` | `true` | 입력 필드를 기록할 수 있는지 정합니다. |
| `captureOutput` | `true` | 출력 필드를 기록할 수 있는지 정합니다. |
| `captureMode` | `"smart"` | `full`, `metadata-only`, `smart` 중 하나입니다. |
| `maxPayloadLength` | `1000` | `smart` 모드에서 보관할 문자열 최대 길이입니다. |

## 수집 모드

| 모드 | 동작 |
| --- | --- |
| `full` | `captureInput`, `captureOutput` 설정이 허용하는 입력과 출력을 보관합니다. |
| `metadata-only` | 입력과 출력 원문을 제거하고 hash, 길이, 지연 시간, 사용량, 오류, 메타데이터를 남깁니다. |
| `smart` | 주요 메타데이터를 남기고 긴 문자열은 줄입니다. 실패와 정책 이벤트는 디버깅에 필요한 정보를 보존합니다. |

```ts
const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
  captureMode: "metadata-only",
});
```

## 전송 옵션

```ts
const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
  delivery: {
    mode: "BACKGROUND",
    timeoutMs: 1000,
    flushOnRunEnd: false,
    flushIntervalMs: 3000,
    batchSize: 50,
    maxQueueSize: 1000,
    overflowStrategy: "DROP_OLDEST",
  },
  circuitBreaker: {
    enabled: true,
    failureThreshold: 3,
    cooldownMs: 30000,
  },
});
```

## 환경 변수

기본 `lynx` export는 환경 변수에서 설정을 읽습니다.

```ts
import { lynx } from "@lynxops/sdk";
```

| 변수 | 설명 |
| --- | --- |
| `LYNX_CLIENT_ID` | 클라이언트 또는 서비스 이름입니다. 기본값은 `local_dev_env`입니다. |
| `LYNX_ENDPOINT` | endpoint를 바꿀 때 사용합니다. 기본값은 `https://api.lynxops.co`입니다. |
| `LYNX_API_KEY` | 이벤트 전송에 사용하는 API key입니다. |
| `LYNX_WORKSPACE_ID` | 기본 workspace ID입니다. |
| `LYNX_AGENT_ID` | 기본 agent ID입니다. |
| `LYNX_SESSION_ID` | `run()`에 session ID를 넘기지 않았을 때 사용할 기본 session ID입니다. |
| `LYNX_SAMPLE_RATE` | `0`부터 `1`까지의 기록 비율입니다. |
| `LYNX_CAPTURE_INPUT` | `true` 또는 `false`입니다. |
| `LYNX_CAPTURE_OUTPUT` | `true` 또는 `false`입니다. |
| `LYNX_CAPTURE_MODE` | `full`, `metadata-only`, `smart` 중 하나입니다. |
| `LYNX_MAX_PAYLOAD_LENGTH` | payload 문자열 최대 길이입니다. |
| `LYNX_APP_VERSION` | 애플리케이션 버전 메타데이터입니다. |
| `LYNX_DEPLOYMENT_ID` | 배포 ID 메타데이터입니다. |
| `LYNX_ENVIRONMENT` | 실행 환경입니다. 없으면 `NODE_ENV`를 사용합니다. |
| `LYNX_POLICY_VERSION` | 기본 정책 버전 메타데이터입니다. |
| `LYNX_DELIVERY_MODE` | `BACKGROUND` 또는 `BLOCKING`입니다. |
| `LYNX_DELIVERY_TIMEOUT_MS` | 전송 timeout입니다. 단위는 millisecond입니다. |
| `LYNX_DELIVERY_FLUSH_ON_RUN_END` | `true` 또는 `false`입니다. |
| `LYNX_DELIVERY_FLUSH_INTERVAL_MS` | 백그라운드 전송 주기입니다. 단위는 millisecond입니다. |
| `LYNX_DELIVERY_BATCH_SIZE` | 한 번에 보낼 이벤트 묶음 크기입니다. |
| `LYNX_DELIVERY_MAX_QUEUE_SIZE` | 메모리 큐에 보관할 최대 이벤트 수입니다. |
| `LYNX_DELIVERY_OVERFLOW_STRATEGY` | `DROP_OLDEST` 또는 `DROP_NEWEST`입니다. |
| `LYNX_CIRCUIT_BREAKER_ENABLED` | `true` 또는 `false`입니다. |
| `LYNX_CIRCUIT_BREAKER_FAILURE_THRESHOLD` | 서킷 브레이커를 열기 전 연속 실패 횟수입니다. |
| `LYNX_CIRCUIT_BREAKER_COOLDOWN_MS` | 다시 전송을 시도하기 전 대기 시간입니다. |

## 보안 메모

설정에 따라 프롬프트, 도구 인자, 모델 응답이 Lynx로 전송될 수 있습니다. API 키, 토큰, 비밀번호, private key 같은 민감정보는 보내지 않는 것이 좋습니다. SDK는 흔한 민감 key를 마스킹하지만, 애플리케이션 코드에서도 민감정보가 들어가지 않게 관리하세요.

원문을 보내고 싶지 않다면 `metadata-only`, `captureInput: false`, `captureOutput: false`를 사용하세요.
