title: "가드레일"
description: "위험한 도구 호출 전에 로컬 정책을 평가합니다."
---

`guardTool()`은 tool 함수를 로컬 policy check로 감쌉니다. Policy는 tool call 전에 실행됩니다. Policy가 차단하면 tool 함수는 실행되지 않습니다.

```ts
import { LynxPolicyError } from "@lynx/sdk";

const refund = lynx.guardTool(
  "refund",
  async ({ amount }: { amount: number }) => {
    return { refunded: amount };
  },
  {
    riskLevel: "HIGH",
    sideEffect: true,
    failureMode: "FAIL_CLOSED",
    beforeCall: ({ input }) => ({
      action: input.amount <= 100 ? "ALLOW" : "BLOCK",
      policyId: "refund-limit",
      reason: "Refund amount is over the limit",
      severity: "HIGH",
    }),
  },
);

try {
  await lynx.run("SupportAgent", () => refund({ amount: 500 }));
} catch (error) {
  if (error instanceof LynxPolicyError) {
    console.log(error.action, error.policyId, error.reason);
  }
}
```

## Policy decision

`beforeCall`은 다음 action을 반환할 수 있습니다.

| Action | 동작 |
| --- | --- |
| `ALLOW` | Tool을 실행합니다. |
| `WARN` | Tool을 실행하고 warning을 기록합니다. |
| `BLOCK` | Tool을 차단하고 `LynxPolicyError`를 던집니다. |
| `REQUIRE_APPROVAL` | Tool을 차단하고 `LynxPolicyError`를 던집니다. |
| `REDACT` | 결정을 기록합니다. Redaction은 policy callback에서 적용한 뒤 tool을 허용하세요. |
| `MODIFY` | 결정을 기록합니다. Modification은 policy callback에서 적용한 뒤 tool을 허용하세요. |

## Failure mode

Policy evaluation 자체가 throw하면 `failureMode`가 동작을 결정합니다.

| Failure mode | 동작 |
| --- | --- |
| `FAIL_OPEN` | Tool call을 허용하고 policy error를 기록합니다. |
| `FAIL_CLOSED` | Tool call을 차단합니다. |
| `REQUIRE_APPROVAL` | `REQUIRE_APPROVAL`로 차단합니다. |

## 서버 가용성

Guardrail 결정은 로컬에서 일어납니다. Lynx 서버가 내려가도 SDK는 policy를 평가합니다. Policy decision telemetry는 queue에 저장되고 나중에 retry됩니다.
