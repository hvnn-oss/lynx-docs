이 문서는 SDK 설치부터 첫 번째 에이전트 실행을 Lynx에서 확인하는 단계까지 안내합니다.

## 언제 사용하나요

TypeScript 서비스에 Lynx를 처음 연결하고, 사용자 입력, 컨텍스트, 판단, 결과가 하나의 세션으로 기록되는지 확인할 때 사용합니다.

## 설정하기

SDK를 설치합니다.

```bash
npm install @lynxops/sdk
```

pnpm이나 Yarn도 사용할 수 있습니다.

```bash
pnpm add @lynxops/sdk
yarn add @lynxops/sdk
```

tracer를 만듭니다.

```ts
import { LynxTracer } from "@lynxops/sdk";

const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
  workspaceId: process.env.LYNX_WORKSPACE_ID,
  environment: "production",
});
```

`endpoint`를 생략하면 SDK는 `https://api.lynxops.co`로 이벤트를 보냅니다.

## 예시

하나의 에이전트 실행을 `run()`으로 감쌉니다. callback 안에서 기록한 이벤트는 같은 실행과 세션에 연결됩니다.

```ts
await lynx.run("SupportAgent", async () => {
  lynx.userInput("환불하고 싶어요", { userId: "usr_123" });
  lynx.setAttributes({ orderId: "order_123" });

  lynx.context("refund-policy", {
    refundWindowDays: 30,
  });

  lynx.decision({
    name: "select_refund_workflow",
    selected: "refund",
    confidence: 0.82,
    reason: "주문이 환불 가능 기간 안에 있습니다",
  });

  lynx.outcome({
    status: "COMPLETED",
    businessStatus: "SUCCEEDED",
    reason: "환불 요청을 접수했습니다",
  });
});
```

## Lynx에서 확인할 수 있는 것

SDK가 이벤트를 보내면 Lynx에서 다음 정보를 확인할 수 있습니다.

- 세션 타임라인
- 실행을 시작한 사용자 입력
- 에이전트가 사용한 컨텍스트
- 에이전트 판단과 이유
- 최종 실행 결과와 비즈니스 결과

## 안전하게 종료하기

서버에서는 종료 hook에서 `shutdown()`을 호출합니다. 서버리스 함수에서는 마지막 전송을 최대한 마무리하고 싶을 때 handler가 끝나기 전에 호출합니다.

```ts
process.on("SIGTERM", () => {
  void lynx.shutdown({ timeoutMs: 1000 });
});
```

<Note>
  기본 전송 방식은 백그라운드 전송입니다. 이벤트 전송에 실패해도 에이전트
  실행 흐름에 예외를 던지지 않습니다.
</Note>
