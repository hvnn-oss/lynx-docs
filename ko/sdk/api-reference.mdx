이 문서는 주요 SDK API를 빠르게 확인하기 위한 레퍼런스입니다. 자세한 예시는 각 기능 문서를 참고하세요.

## `LynxTracer`

`LynxTracer`는 이벤트를 기록하고, 현재 실행 context를 관리하며, 이벤트 묶음을 Lynx로 전송합니다.

```ts
import { LynxTracer } from "@lynxops/sdk";

const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
});
```

## `lynx`

`lynx`는 환경 변수에서 설정을 읽는 기본 tracer입니다.

```ts
import { lynx } from "@lynxops/sdk";
```

간단한 애플리케이션에서는 이 기본 인스턴스를 사용할 수 있습니다. 프로세스, tenant, test마다 설정을 명확히 분리해야 한다면 `new LynxTracer()`를 사용하세요.

## 실행 메서드

### `run(agentName, callback)`

코드를 Lynx 실행 context 안에서 실행합니다.

```ts
await lynx.run("SupportAgent", async () => {
  lynx.userInput("비밀번호를 재설정하고 싶어요");
});
```

### `run(options, callback)`

구조화된 실행 메타데이터를 함께 넘깁니다.

```ts
await lynx.run(
  {
    agentName: "SupportAgent",
    workspaceId: "workspace_123",
    agentId: "agent_123",
    sessionId: "session_123",
  },
  async () => {
    // Agent work
  },
);
```

## Semantic helper

| Method | 목적 |
| --- | --- |
| `userInput(input, metadata?)` | 사용자 입력을 기록합니다. |
| `decision(reasonOrDecision, options?)` | 에이전트 판단을 기록합니다. |
| `context(labelOrData, dataOrMetadata?, metadata?)` | 검색되었거나 구성된 컨텍스트를 기록합니다. |
| `memory(operation, data, metadata?)` | 메모리 접근을 기록합니다. |
| `outcome(options)` | 최종 기술 결과와 비즈니스 결과를 기록합니다. |
| `setAttributes(attributes)` | 현재 실행의 이후 이벤트에 공통 속성을 붙입니다. |
| `annotate(label, payload)` | 커스텀 breadcrumb를 기록합니다. |
| `log(label, payload)` | 커스텀 context 이벤트를 기록합니다. |

## LLM과 도구 helper

| Method | 목적 |
| --- | --- |
| `instrumentLLM(client, options?)` | LLM 클라이언트 객체를 감쌉니다. |
| `instrumentTool(name, fn, metadata?)` | 도구 함수를 감쌉니다. |
| `guardTool(name, fn, options?)` | 도구 함수를 로컬 정책 평가와 함께 감쌉니다. |

## 전송 helper

| Method | 목적 |
| --- | --- |
| `flush()` | 큐에 쌓인 이벤트를 지금 전송합니다. |
| `shutdown(options?)` | 타이머를 정리하고 종료 전 마지막 전송을 시도합니다. |
| `getStatus()` | 로컬 큐, 서킷 브레이커, 전송 상태를 반환합니다. |

## `LynxPolicyError`

`guardTool()`이 도구 호출을 차단하면 `LynxPolicyError`를 던집니다.

```ts
import { LynxPolicyError } from "@lynxops/sdk";

if (error instanceof LynxPolicyError) {
  console.log(error.action);
  console.log(error.policyId);
  console.log(error.reason);
}
```

## 설정 옵션

| Option | 설명 |
| --- | --- |
| `clientId` | 필수 클라이언트 또는 서비스 이름입니다. |
| `endpoint` | Lynx API endpoint입니다. 기본값은 `https://api.lynxops.co`입니다. |
| `workspaceId` | 이벤트에 붙일 기본 workspace ID입니다. |
| `agentId` | 이벤트에 붙일 기본 agent ID입니다. |
| `apiKey` | 이벤트 전송에 사용하는 API key입니다. |
| `appVersion` | 애플리케이션 버전 메타데이터입니다. |
| `deploymentId` | 배포 ID 메타데이터입니다. |
| `environment` | 실행 환경 메타데이터입니다. |
| `policyVersion` | 기본 정책 버전 메타데이터입니다. |
| `sampleRate` | `0`부터 `1`까지의 기록 비율입니다. |
| `captureInput` | 입력 필드를 기록할 수 있는지 정합니다. |
| `captureOutput` | 출력 필드를 기록할 수 있는지 정합니다. |
| `captureMode` | `full`, `metadata-only`, `smart` 중 하나입니다. |
| `maxPayloadLength` | `smart` 모드에서 보관할 문자열 최대 길이입니다. |
| `delivery.mode` | `BACKGROUND` 또는 `BLOCKING`입니다. |
| `delivery.timeoutMs` | 한 번의 전송 요청 timeout입니다. |
| `delivery.flushOnRunEnd` | `run()` callback이 끝난 뒤 flush를 시도할지 정합니다. |
| `delivery.flushIntervalMs` | 백그라운드 전송 주기입니다. |
| `delivery.batchSize` | 이벤트 묶음 크기입니다. |
| `delivery.maxQueueSize` | 메모리에 보관할 최대 이벤트 수입니다. |
| `delivery.overflowStrategy` | `DROP_OLDEST` 또는 `DROP_NEWEST`입니다. |
| `circuitBreaker.enabled` | 전송용 서킷 브레이커를 사용할지 정합니다. |
| `circuitBreaker.failureThreshold` | 서킷 브레이커를 열기 전 연속 실패 횟수입니다. |
| `circuitBreaker.cooldownMs` | 다시 전송을 시도하기 전 대기 시간입니다. |
