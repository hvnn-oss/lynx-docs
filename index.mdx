Lynx SDK helps you understand why an AI agent behaved a certain way. It captures execution context, LLM calls, tool usage, policy decisions, memory access, and outcomes, then sends telemetry to your Lynx server.

Use Lynx when you need production observability for AI services, not just logs.

## What Lynx captures

- Agent run identity, session identity, and workspace metadata
- User input and structured context
- Agent decisions and reasons
- LLM calls, model metadata, latency, token usage, and errors
- Tool calls, tool results, side effects, and risk metadata
- Local policy evaluations and guardrail activations
- Memory access and final session outcome

## What happens if Lynx is unavailable?

Your AI service keeps running by default. The SDK queues telemetry in memory, retries failed sends, and opens a circuit breaker after repeated delivery failures. Guardrail decisions still happen locally.

<Tip>
  Lynx telemetry is designed to be observable, not blocking. You can opt into blocking flush behavior when your environment requires it.
</Tip>
