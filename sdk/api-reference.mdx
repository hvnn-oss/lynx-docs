This page summarizes the primary SDK APIs. Use the feature guides for longer examples.

## `LynxTracer`

`LynxTracer` records events, manages the active run context, and delivers event batches to Lynx.

```ts
import { LynxTracer } from "@lynxops/sdk";

const lynx = new LynxTracer({
  clientId: "support-api",
  apiKey: process.env.LYNX_API_KEY,
});
```

## `lynx`

`lynx` is a default tracer configured from environment variables.

```ts
import { lynx } from "@lynxops/sdk";
```

Use this for simple applications. Use `new LynxTracer()` when you need explicit configuration per process, tenant, or test.

## Run methods

### `run(agentName, callback)`

Runs code inside a Lynx context.

```ts
await lynx.run("SupportAgent", async () => {
  lynx.userInput("Help me reset my password");
});
```

### `run(options, callback)`

Runs code with structured run metadata.

```ts
await lynx.run(
  {
    agentName: "SupportAgent",
    workspaceId: "workspace_123",
    agentId: "agent_123",
    sessionId: "session_123",
  },
  async () => {
    // Agent work
  },
);
```

## Semantic helpers

| Method | Purpose |
| --- | --- |
| `userInput(input, metadata?)` | Records user input. |
| `decision(reasonOrDecision, options?)` | Records an agent decision. |
| `context(labelOrData, dataOrMetadata?, metadata?)` | Records retrieved or constructed context. |
| `memory(operation, data, metadata?)` | Records memory access. |
| `outcome(options)` | Records the final technical and business outcome. |
| `setAttributes(attributes)` | Attaches attributes to later events in the current run. |
| `annotate(label, payload)` | Records a custom breadcrumb. |
| `log(label, payload)` | Records a custom context event. |

## LLM and tool helpers

| Method | Purpose |
| --- | --- |
| `instrumentLLM(client, options?)` | Wraps an LLM client object. |
| `instrumentTool(name, fn, metadata?)` | Wraps a tool function. |
| `guardTool(name, fn, options?)` | Wraps a tool with local policy evaluation. |

## Delivery helpers

| Method | Purpose |
| --- | --- |
| `flush()` | Sends queued events now. |
| `shutdown(options?)` | Clears timers and performs a best-effort final flush. |
| `getStatus()` | Returns local queue, circuit breaker, and delivery state. |

## `LynxPolicyError`

`guardTool()` throws `LynxPolicyError` when a tool call is blocked.

```ts
import { LynxPolicyError } from "@lynxops/sdk";

if (error instanceof LynxPolicyError) {
  console.log(error.action);
  console.log(error.policyId);
  console.log(error.reason);
}
```

## Configuration options

| Option | Description |
| --- | --- |
| `clientId` | Required client or service identifier. |
| `endpoint` | Lynx API endpoint. Defaults to `https://api.lynxops.co`. |
| `workspaceId` | Default workspace ID attached to events. |
| `agentId` | Default agent ID attached to events. |
| `apiKey` | API key for event ingestion. |
| `appVersion` | Application version metadata. |
| `deploymentId` | Deployment ID metadata. |
| `environment` | Runtime environment metadata. |
| `policyVersion` | Default policy version metadata. |
| `sampleRate` | Sampling ratio from `0` to `1`. |
| `captureInput` | Whether input fields may be recorded. |
| `captureOutput` | Whether output fields may be recorded. |
| `captureMode` | `full`, `metadata-only`, or `smart`. |
| `maxPayloadLength` | Maximum string length kept in `smart` mode. |
| `delivery.mode` | `BACKGROUND` or `BLOCKING`. |
| `delivery.timeoutMs` | Timeout for one delivery request. |
| `delivery.flushOnRunEnd` | Whether `run()` tries to flush after the callback finishes. |
| `delivery.flushIntervalMs` | Background flush interval. |
| `delivery.batchSize` | Event batch size. |
| `delivery.maxQueueSize` | Maximum queued event count in memory. |
| `delivery.overflowStrategy` | `DROP_OLDEST` or `DROP_NEWEST`. |
| `circuitBreaker.enabled` | Enables delivery circuit breaker protection. |
| `circuitBreaker.failureThreshold` | Consecutive failures before opening the breaker. |
| `circuitBreaker.cooldownMs` | Cooldown before another send attempt. |
